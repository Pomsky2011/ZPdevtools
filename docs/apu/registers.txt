# ZeroPoint APU Register Reference

## Register Overview

The APU has 256 general-purpose 8-bit registers plus several special-purpose registers.

### General Purpose Registers
- **R0 (X)** (8-bit) - Primary accumulator/index register
- **R1 (Y)** (8-bit) - Secondary accumulator/index register
- **R2-R255** (8-bit each) - Extended general purpose registers

Note: Arithmetic instructions (NOR, AND, ADD, SUB, ADC, SBC, XOR, NOR) encode source
operands in 1 bit (0=R0/X, 1=R1/Y). Destination can be any register R0-R255 or memory.
Extended registers R2-R255 are used as destinations or via STA/STR/LDA.

### Special Purpose Registers
- **PC** (Program Counter) - 16-bit, tracks current instruction address
- **SP** (Stack Pointer) - 16-bit, hardware stack pointer
- **RP** (ROM Page) - 8-bit, high byte for jump target addressing
- **DP** (Data Page) - 8-bit, high byte for memory addressing
- **DB** (Data Byte) - 8-bit, secondary memory addressing byte
- **BF** (Byte Flip) - 1-bit, selects high/low byte in word operations
- **FLAGS** - 4-bit status flags register (0bzglc)

## Detailed Register Descriptions

### R0 / X Register (8-bit)

**Purpose:** Primary general-purpose register. Encoded as source operand 0 in arithmetic.

**Can Be Used In:**
- NOR, AND, ADD, SUB, ADC, SBC, XOR (source/destination)
- STA, STR / LDA (data storage/loading)
- SCR (load immediate constant)
- ZOR, ZOA (zero out)
- BEQ, BNE, BLT, BGT (conditional branches)
- CME, CMN, CML, CMG (compare-and-jump)
- JNZ, JZ (conditional jumps)
- MOV, EXC (register transfer/exchange)
- INC, DEC (increment/decrement)
- PUX, POX (push/pop to hardware stack)

**Common Use Cases:**
```
LDA X, 42         ; X = 42
ADD X, Y, X       ; X = X + Y
STA X, $50        ; Store X to [DP:$50]
JNZ X, $10        ; Jump to [RP:$10] if X != 0
```

---

### R1 / Y Register (8-bit)

**Purpose:** Secondary general-purpose register. Encoded as source operand 1 in arithmetic.

**Can Be Used In:**
- All same instructions as R0/X
- Typically used as the second operand in binary operations

**Common Use Cases:**
```
LDA X, 10
LDA Y, 20
ADD X, Y, X       ; X = 30
BEQ X, Y, $50     ; Branch to [RP:$50] if X == Y
```

---

### R2-R255 Extended Registers (8-bit each)

**Purpose:** General purpose storage. Not addressable as arithmetic sources directly,
but usable as destinations and via STA/STR.

**Conventions (suggested):**
- R0-R3:   Temporaries / function arguments / return values
- R4-R15:  Callee-saved locals
- R16-R31: General locals
- R32-R255: Extended storage / large arrays

**Usage:**
```
LDA X, 42
STA X, $20        ; Store X into [DP:$20] (accessible as R-indexed memory)
STR $20, X        ; Load [DP:$20] back into X
```

---

### PC - Program Counter (16-bit)

**Purpose:** Address of the currently executing instruction.

**Characteristics:**
- 16-bit (full 64 KiB address space)
- Increments by 2 after each instruction fetch
- Modified by all jump/branch/call/return instructions

**Modification Instructions:**
- `JMP $ZZ` / `JMP DP` — unconditional jump
- `JSR $ZZ` / `JSR DP` — call (pushes return address)
- `RET` — return (pops return address)
- `JNZ X, $ZZ` — jump if R0/R1 != 0
- `JZ X, $ZZ` — jump if R0/R1 == 0
- `BEQ/BNE/BLT/BGT X, Y, $ZZ` — compare-and-branch

**Executable Regions:**
```
$0100-$0FFF  RAM (self-modifying code)
$8000-$8FFF  BIOS ROM
$D000-$FFFF  AROM (banked)
```

---

### SP - Stack Pointer (16-bit)

**Purpose:** Hardware stack pointer. Manages the call stack and push/pop operations.

**Characteristics:**
- 16-bit register
- Grows downward (SP decrements on push, increments on pop)
- Managed automatically by push/pop and call/return instructions
- Can be manually adjusted with `INC SP` / `DEC SP`

**Instructions that use SP:**
- `PUX` — push R0 (X) onto stack
- `PUY` — push R1 (Y) onto stack
- `PUDP` — push DP onto stack
- `POX` — pop from stack into R0 (X)
- `POY` — pop from stack into R1 (Y)
- `PODP` — pop from stack into DP
- `JSR $ZZ` / `JSR DP` — pushes return address (2 bytes)
- `RET` — pops return address (2 bytes)
- `INC SP` / `DEC SP` — manual SP adjustment
- `BSP $XX` — set SP low byte

**Set By:**
```
BSP $FF       ; SP low byte = $FF (initial stack setup)
INC SP        ; SP++
DEC SP        ; SP--
```

**Stack Layout Example:**
```
; Stack grows down from $01FF
BSP $FF       ; SP = $xxFF
SRP $01
; Stack now starts at $01FF
```

---

### RP - ROM Page Register (8-bit)

**Purpose:** High byte of jump target addresses.

**Used In:**
- `JMP $ZZ` (Y=0) → jumps to $RPZZ
- `JNZ X, $ZZ` → jumps to $RPZZ if X != 0
- `JZ X, $ZZ` → jumps to $RPZZ if X == 0
- `JSR $ZZ` → calls $RPZZ (pushes return address)
- `BEQ/BNE/BLT/BGT X, Y, $ZZ` → conditional branch to $RPZZ

**Set By:**
```
SRP $D0       ; RP = $D0
```

**Example:**
```
SRP $80       ; Set RP to BIOS region
JMP $10       ; Jump to $8010

SRP $D0       ; Set RP to AROM region
BEQ X, Y, $50 ; If X == Y, jump to $D050
```

**Memory Page Mapping:**
```
RP=$00 → Jumps to $00XX (MMP registers)
RP=$01 → Jumps to $01XX (RAM)
RP=$10 → Jumps to $10XX (DEF88186 Input)
RP=$80 → Jumps to $80XX (BIOS)
RP=$D0 → Jumps to $D0XX (AROM)
```

---

### DP - Data Page Register (8-bit)

**Purpose:** High byte for memory data addresses.

**Used In:**
- `STA X, $YY` → stores to $DPYY
- `STR $YY, X` → loads from $DPYY
- `NOR/AND/ADD/SUB/ADC/SBC/XOR` with memory destination → result to $DPZZ
- `ZOA $XX` → zeros $DPXX
- `JMP DP` / `JSR DP` → jumps to $DPDB (combined with DB)

**Set By:**
```
SDP $30             ; DP = $30 (immediate)
SDP b, $XX          ; DP[b] = $XX (set high or low nibble)
PODP                ; Pop DP from stack
```

**Common Values:**
```
SDP $00  ; MMP control registers
SDP $01  ; General RAM
SDP $10  ; DEF88186 Input
SDP $18  ; DEF88186 Output
SDP $30  ; ARAM
SDP $70  ; STL
SDP $80  ; BIOS (read-only)
SDP $90  ; SST
SDP $D0  ; AROM (read-only)
```

**Example:**
```
SDP $10       ; DP = $10
LDA X, 42
STA X, $50    ; Store 42 to $1050

SDP $90       ; DP = $90 (SST region)
STR $00, X    ; Load from $9000 into X
```

**Notes:**
- Push/pop with `PUDP`/`PODP` for context switches
- Set once and access multiple offsets in same page

---

### DB - Data Byte Register (8-bit)

**Purpose:** Low byte offset for `JMP DP` / `JSR DP` indirect jumps.

**Used In:**
- `JMP DP` → jumps to address formed as ($DP << 8) | $DB
- `JSR DP` → calls address ($DP << 8) | $DB (pushes return address)

**Set By:**
```
SDB $50       ; DB = $50
```

**Example:**
```
SDP $D0       ; DP = $D0
SDB $40       ; DB = $40
JMP DP        ; Jump to $D040
```

**Notes:**
- DB is secondary to DP for indirect jumps
- Not used for STA/STR addressing (those use $DPYY directly)

---

### BF - Byte Flip Register (1-bit)

**Purpose:** Selects high or low byte in 16-bit word memory operations.

**Values:**
- 0 = Low byte (bits 7-0)
- 1 = High byte (bits 15-8)

**Used In:**
- `STA X, $YY` — which byte of $DPYY to write
- `STR $YY, X` — which byte of $DPYY to read
- Arithmetic instructions with memory destination

**Set By:**
```
SBF 0         ; BF = 0 (low byte)
SBF 1         ; BF = 1 (high byte)
```

**Example - Reading a 16-bit Value:**
```
SDP $10
SBF 0
STR $50, X    ; X = low byte of $1050
SBF 1
STR $50, Y    ; Y = high byte of $1050
```

**Example - Writing a 16-bit Value:**
```
SDP $20
LDA X, $CD    ; Low byte
LDA Y, $AB    ; High byte
SBF 0
STA X, $30    ; Write low byte to $2030
SBF 1
STA Y, $30    ; Write high byte to $2030
; $2030 now contains $ABCD
```

---

### FLAGS - Status Flags Register (4-bit)

**Purpose:** Condition codes set by arithmetic/compare instructions.

**Bit Layout:** `0b0zglc` (4 bits)
```
Bit 3 (0x08): Z = Zero flag      — result was zero
Bit 2 (0x04): G = Greater flag   — left operand > right
Bit 1 (0x02): L = Less flag      — left operand < right
Bit 0 (0x01): C = Carry flag     — arithmetic carry/borrow
```

**Set By:**
| Instruction | Flags Affected         |
|-------------|------------------------|
| ADD X, Y    | Z, C                   |
| SUB X, Y    | Z, C, G, L             |
| ADC X, Y    | Z, C                   |
| SBC X, Y    | Z, C, G, L             |
| CME/CMN/CML/CMG | Z, G, L           |
| BEQ/BNE/BLT/BGT | Z, G, L (before branch) |
| SF n        | Sets specific flag bit  |
| CF n        | Clears specific flag bit |
| SFR reg     | FLAGS = reg & 0x0F      |
| STF reg     | reg = FLAGS             |

**Read By:**
- `BEQ/BNE/BLT/BGT` — branch based on Z/G/L
- `ADC/SBC` — reads C flag to include carry
- `STF X` — reads FLAGS into X for manual inspection

**Flag Instructions:**
```
SF 0          ; Set carry flag (FLAGS |= 0x01)
SF 1          ; Set less flag
SF 2          ; Set greater flag
SF 3          ; Set zero flag
CF 0          ; Clear carry flag
CF 1          ; Clear less flag
SFR X         ; Load FLAGS from X (FLAGS = X & 0x0F)
STF X         ; Store FLAGS into X
```

**Example - Carry Arithmetic:**
```
; Add 16-bit values: (R2:R3) + (R4:R5)
LDA X, R3     ; not direct, use STA/STR
; assume X = low byte, Y = low byte of second
ADD X, Y, X   ; X = low + low (sets C on overflow)
STF Y         ; save flags
; now do high bytes
LDA X, R2_high
LDA Y, R4_high
ADC X, Y, X   ; X = high + high + carry
```

**Example - Manual Flag Check:**
```
SUB X, Y, X   ; X = X - Y, sets Z/G/L/C
STF Y         ; Y = flags
AND Y, X, Y   ; check zero: Y &= 0x08 (FLAG_Z)
JNZ Y, zero_case
```

## Register Usage Conventions

### Function Calling Convention

**Arguments (caller-provided):**
- R0 (X), R1 (Y) for up to 2 single-byte parameters
- Additional args pushed on stack before call

**Return values:**
- R0 (X) for single byte
- R0 (X) + R1 (Y) for two bytes

**Caller-saved (caller must preserve if needed):**
- R0 (X), R1 (Y), FLAGS

**Callee-saved (callee must preserve):**
- RP, DP, DB, BF
- R4-R15 (by convention)

**Call/Return Example:**
```
; Caller
LDA X, 5      ; arg0 = 5
LDA Y, 3      ; arg1 = 3
SRP $D0
JSR $40       ; call multiply at $D040
; X = result

; Callee (multiply: X * Y)
multiply:
  PUY           ; save Y across loop
  ; ... compute ...
  POY           ; restore Y
  RET
```

### Stack Operations

The APU has hardware stack instructions. No manual SP tracking needed:

```
PUX           ; push X
PUY           ; push Y
PUDP          ; push DP
JSR $XX       ; call (auto-pushes return address)
RET           ; return (auto-pops return address)
POX           ; pop into X
POY           ; pop into Y
PODP          ; pop into DP
```

**Saving/restoring a register across a call:**
```
PUX           ; save X
JSR $XX       ; call function (may clobber X)
POX           ; restore X
```

## Register State on Reset

On APU reset:
- PC: $8000 (BIOS entry point)
- SP: $01FF (top of RAM stack area)
- RP, DP, DB, BF: $00, $00, $00, 0
- FLAGS: $00 (all clear)
- R0-R255: undefined, should be initialized

**Recommended Boot Sequence:**
```
start:
  SRP $D0       ; RP → AROM region
  SDP $01       ; DP → RAM
  SDB $00       ; DB = 0
  SBF 0         ; BF = low byte
  CF 0          ; clear carry
  CF 1          ; clear less
  CF 2          ; clear greater
  CF 3          ; clear zero
  ZOR X         ; clear X
  ZOR Y         ; clear Y
  ; ... initialize RAM, then jump to main
```
