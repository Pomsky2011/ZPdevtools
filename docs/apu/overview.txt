# ZeroPoint APU Architecture Overview

## Introduction

The ZeroPoint Audio Processing Unit (APU) is an 8-bit RISC processor designed
specifically for real-time audio processing and music mixing. It runs alongside
the main CPU to handle all audio-related tasks.

## Core Specifications

- **Architecture**: 8-bit RISC
- **Clock Speed**: 4 MHz (executes every 16th master clock cycle)
- **Cycles Per Instruction (CPI)**: 4
- **Effective Performance**: 1.0 MIPS (Million Instructions Per Second)
- **Instruction Size**: 16-bit fixed width
- **Instruction Format**: 5-bit opcode + 11-bit operands

## System Clock Synchronization

The APU runs at 4 MHz, synchronized to the 64 MHz master clock. It executes only on
master cycle 15 of each 16-cycle pattern:

- **Master clock**: 64 MHz (PPU pixel clock)
- **APU clock**: 4 MHz (every 16 master cycles)
- **Execution pattern**: Master cycles 0-14 (idle), 15 (APU executes)

This deterministic pattern ensures perfect synchronization with other system components.

## Performance Characteristics

### Clock Budget
- Total cycles per second: 4,000,000
- Instructions per second: 1,000,000 (at 4 CPI average)
- At 60 Hz audio: ~16,666 instructions per frame
- At 48 kHz sample rate: ~20.83 instructions per sample

### Audio Frame Budget (60 Hz)
```
Frame time: 16.67 ms
Cycles per frame: 66,666
Instructions per frame: 16,666 (at 4 CPI)
Samples per frame (48 kHz): 800
Instruction budget per sample: ~20 instructions
```

## Major Subsystems

### 1. CPU Core (UAS - Universal Audio System)
- 8-bit register-based processor
- 5 general-purpose 8-bit registers (X, Y, and 3 others usable in operations)
- Dedicated registers for memory management (DP, RP, DB, BF)
- 16-bit program counter
- Support for loops, functions, and conditional branching

### 2. MMP (Music Mixing Processor)
- Hardware mixing for 16 channels with stereo panning
- Per-channel pitch control (resampling, fixed-point 12-bit fraction)
- Per-channel volume control (0=silent, 255=max)
- Per-channel pan (0=full L, 128=center, 255=full R)
- Linear interpolation for pitch-shifted playback

### 3. SST (Sample Storage/Streaming)
- Sample block format with loop support
- 12-byte sample blocks per entry
- Clamping system for dynamic range compression/expansion
- Loop points and one-shot playback modes
- 4-byte headers per block with extensive control flags

### 4. DEF88186 Interface
- Bidirectional communication with main CPU
- 2 KiB input buffer ($1000-$17FF)
- 2 KiB output buffer ($1800-$1FFF)
- Allows CPU to control APU and receive status

## Memory Organization

Total addressable space: 64 KiB (65,536 bytes)
Note: Actually uses 32,768 words (16-bit) but addressed as 64 KiB

### Memory Regions
```
$0000-$00FF   MMP Control Registers + I/O (256 bytes)
$0100-$0FFF   General Purpose RAM (3.75 KiB)
$1000-$17FF   DEF88186 Input from CPU (2 KiB)
$1800-$1FFF   DEF88186 Output to CPU (2 KiB)
$2000-$2FFF   MMP ExIO (Extra I/O) (4 KiB)
$3000-$6FFF   ARAM (Audio RAM) (16 KiB)
$7000-$7FFF   STL (Sample Table List) (4 KiB)
$8000-$8FFF   BIOS (4 KiB)
$9000-$CFFF   SST (Sample Storage) (16 KiB)
$D000-$FFFF   AROM (Audio ROM, banked) (12 KiB visible)
```

### Banking System
- **AROM**: 448 KiB total, organized into 24 KiB banks
- **Bank Switch**: Use RBC (ROM Bank Change) instruction
- **Bank Count**: 18 banks × 24 KiB each
- **Visible Window**: $D000-$FFFF (12 KiB at a time)

Total ROM capacity: 448 KiB (Audio ROM) + 64 KiB (SST) = 512 KiB

## Programming Model

### Instruction Categories
1. **Control Flow**: JMP, JNZ, conditional branches, loops, functions
2. **Arithmetic/Logic**: NOR, AND, ADD, SUB
3. **Memory Access**: STA (store), STR (load), WRH/WRL (write immediate)
4. **Register Operations**: SCR (set constant), ZOR (zero), SBF (byte flip)
5. **I/O**: IOO (output), IOI (input), IBC (I/O bank change)
6. **Special**: NOP with cycle stall, SRP/SDP/SDB (set page registers)

### Key Features
- **Loops**: Hardware loop support with 16 unique loop IDs
- **Functions**: Callable function system with CFS/CFE/CCF
- **Banking**: Separate ROM and I/O banks for extended addressing
- **Word Operations**: BF (Byte Flip) register allows 16-bit operations via high/low byte selection
- **Conditional Jumps**: Multiple addressing modes (absolute, relative, offset)

## Audio Pipeline

```
[Main CPU] <--DEF88186--> [APU Core]
                             |
                             v
                        [MMP Mixer]
                        16 channels
                     (pitch, vol, pan)
                             |
                    +--------+--------+
                    |                 |
               [Left Mix]       [Right Mix]
            pan-weighted       pan-weighted
                    |                 |
                    v                 v
                [Left Out]       [Right Out]
```

## Typical Use Cases

### 1. Music Playback
- Load sample data into SST region
- Configure MMP channels with STL pointers
- Set pitch, volume, pan, and loop points
- APU handles mixing and stereo panning automatically

### 2. Sound Effects
- Use short one-shot samples
- Dynamically allocate channels
- Adjust pitch for variations
- Layer multiple channels for complex effects

### 3. Real-time Synthesis
- Use APU core for waveform generation
- Write directly to ARAM
- Point MMP channels to generated data
- Update continuously for synthesis

## Performance Optimization Tips

1. **Use Hardware Features**: Let MMP handle mixing rather than doing it in software
2. **Minimize Memory Access**: Keep frequently-used data in registers
3. **Leverage Loops**: Hardware loops are more efficient than conditional jumps
4. **Batch Operations**: Use callable functions for repeated code sequences
5. **Banking Strategy**: Organize code to minimize bank switches
6. **Sample Reuse**: Use pitch shifting to create variations from one sample
