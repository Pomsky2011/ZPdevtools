================================================================================
DEF88186 C Compiler - Cross-Compilation Guide (macOS/Linux to DOS/Windows)
================================================================================

OVERVIEW
================================================================================

This guide shows how to cross-compile the def88186 compiler from macOS or
Linux to create executables that run on DOS or Windows systems.

TARGET OPTIONS
================================================================================

You can create executables for:

1. MS-DOS (16-bit Real Mode)
   - Using DJGPP cross-compiler
   - Creates .EXE files that run on DOS 4.0+
   - Requires DPMI (included with DJGPP)

2. MS-DOS (32-bit Protected Mode via DJGPP)
   - Using DJGPP cross-compiler
   - Creates 32-bit DOS executables
   - Runs on DOS 5.0+ with DPMI

3. Windows (32-bit)
   - Using MinGW-w64 (i686)
   - Creates Win32 .EXE files
   - Runs on Windows 95+ and modern Windows

4. Windows (64-bit)
   - Using MinGW-w64 (x86_64)
   - Creates Win64 .EXE files
   - Runs on Windows XP 64-bit and later

OPTION 1: DJGPP Cross-Compiler (DOS 32-bit)
================================================================================

DJGPP creates TRUE DOS executables using DOS/4GW extender.

Installation on macOS:

1. Install via Homebrew:
   $ brew tap tavianator/tap
   $ brew install djgpp

   OR build from source:
   $ git clone https://github.com/andrewwutw/build-djgpp
   $ cd build-djgpp
   $ ./build-djgpp.sh --prefix=/usr/local/djgpp

2. Add to PATH:
   $ export PATH=/usr/local/djgpp/bin:$PATH

3. Cross-compile:
   $ cd def88cc
   $ i586-pc-msdosdjgpp-gcc -std=c89 -O2 -o def88186.exe \
       main.c ast.c codegen.c preproc.c lexerman.c parser.c

4. Result:
   - def88186.exe - DOS 32-bit executable
   - Runs on any DOS system with DPMI (included)
   - File size: ~200-300 KB

Installation on Linux:

1. Debian/Ubuntu:
   $ sudo apt-get install djgpp djgpp-dev

2. Fedora/RHEL:
   $ sudo dnf install djgpp-gcc djgpp-binutils

3. Build from source (all distros):
   $ git clone https://github.com/andrewwutw/build-djgpp
   $ cd build-djgpp
   $ ./build-djgpp.sh --prefix=$HOME/djgpp

4. Cross-compile (same as macOS above)

OPTION 2: MinGW-w64 (Windows Executables)
================================================================================

MinGW-w64 creates Windows .EXE files that also work in DOS via DOS extenders.

Installation on macOS:

1. Install via Homebrew:
   $ brew install mingw-w64

2. For 32-bit Windows (works on DOS with HX DOS Extender):
   $ i686-w64-mingw32-gcc -std=c89 -O2 -o def88186.exe \
       main.c ast.c codegen.c preproc.c lexerman.c parser.c

3. For 64-bit Windows:
   $ x86_64-w64-mingw32-gcc -std=c89 -O2 -o def88186.exe \
       main.c ast.c codegen.c preproc.c lexerman.c parser.c

Installation on Linux:

1. Debian/Ubuntu:
   $ sudo apt-get install mingw-w64

2. Fedora/RHEL:
   $ sudo dnf install mingw64-gcc mingw32-gcc

3. Arch Linux:
   $ sudo pacman -S mingw-w64-gcc

RECOMMENDED: DJGPP for Authentic DOS
================================================================================

For the most authentic retro experience, use DJGPP to create TRUE DOS
executables:

Complete Build Script (macOS):

#!/bin/bash
# Build def88186 for MS-DOS using DJGPP cross-compiler

# Set DJGPP paths
export DJGPP_PREFIX=/usr/local/djgpp
export PATH=$DJGPP_PREFIX/bin:$PATH
export CC=i586-pc-msdosdjgpp-gcc

# Compiler flags
CFLAGS="-std=c89 -O2 -Wall -Wno-unused-function"

# Compile sources
$CC $CFLAGS -c main.c
$CC $CFLAGS -c ast.c
$CC $CFLAGS -c codegen.c
$CC $CFLAGS -c preproc.c
$CC $CFLAGS -c lexerman.c
$CC -c parser.c -w

# Link
$CC -o def88186.exe main.o ast.o codegen.o preproc.o lexerman.o parser.o

# Strip to reduce size
i586-pc-msdosdjgpp-strip def88186.exe

echo "Build complete!"
echo "File: def88186.exe"
echo "Size: $(ls -lh def88186.exe | awk '{print $5}')"
echo ""
echo "Transfer to DOS and run:"
echo "  C:\\> def88186 test.c -o test.asm"

ALTERNATIVE: Build Native, Run via DOSBox
================================================================================

The simplest approach for testing:

1. Build natively on macOS/Linux:
   $ make

2. Run via DOSBox-X or PCem:
   $ dosbox-x
   C:\> mount c ~/def88cc
   C:\> c:
   C:\> def88186 test.c -o test.asm

This works because the native macOS/Linux binary can run in DOSBox's
emulated x86 environment... wait, no it can't. You DO need cross-compilation.

MAKEFILE SUPPORT
================================================================================

Add cross-compilation targets to Makefile:

# Cross-compilation targets
DJGPP_CC = i586-pc-msdosdjgpp-gcc
MINGW32_CC = i686-w64-mingw32-gcc
MINGW64_CC = x86_64-w64-mingw32-gcc

# Build for DOS (DJGPP)
dos:
	$(DJGPP_CC) -std=c89 -O2 -o def88186.exe \
		main.c ast.c codegen.c preproc.c lexerman.c parser.c
	i586-pc-msdosdjgpp-strip def88186.exe

# Build for Windows 32-bit
win32:
	$(MINGW32_CC) -std=c89 -O2 -o def88186.exe \
		main.c ast.c codegen.c preproc.c lexerman.c parser.c
	i686-w64-mingw32-strip def88186.exe

# Build for Windows 64-bit
win64:
	$(MINGW64_CC) -std=c89 -O2 -o def88186.exe \
		main.c ast.c codegen.c preproc.c lexerman.c parser.c
	x86_64-w64-mingw32-strip def88186.exe

Usage:
  $ make dos      # Create DOS executable
  $ make win32    # Create Windows 32-bit
  $ make win64    # Create Windows 64-bit

FILE SIZE COMPARISON
================================================================================

Typical executable sizes:

Native macOS (ARM64):           ~80 KB
Native Linux (x86_64):          ~85 KB
DJGPP DOS (32-bit):             ~250 KB (includes DPMI stub)
MinGW Win32:                    ~120 KB
MinGW Win64:                    ~130 KB

DJGPP is larger due to DOS extender stub, but it's a TRUE DOS executable!
