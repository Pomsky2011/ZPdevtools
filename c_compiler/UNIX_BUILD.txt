BUILDING ON UNIX WORKSTATION
================================================================================

Quick Start:
  $ cd def88cc
  $ make CC=gcc                   # Use GCC
  $ make CC=cc                    # Use native compiler

Build Steps:

1. Generate parser (one-time):
   $ bison -d -o parser.c parser.y
   OR
   $ yacc -d parser.y
   $ mv y.tab.c parser.c
   $ mv y.tab.h parser.h

2. Compile all sources:
   $ gcc -std=c89 -c main.c
   $ gcc -std=c89 -c ast.c
   $ gcc -std=c89 -c codegen.c
   $ gcc -std=c89 -c preproc.c
   $ gcc -std=c89 -c lexerman.c
   $ gcc -c parser.c            # No strict flags for generated code

3. Link:
   $ gcc -o def88186 main.o ast.o codegen.o preproc.o lexerman.o parser.o

4. Test:
   $ ./def88186 test.c -o test.asm

PLATFORM-SPECIFIC NOTES
================================================================================

SunOS / Solaris:
  - Use /usr/ucb/cc for BSD compatibility
  - GCC preferred for ANSI C compliance
  - Add -I/usr/include to find headers
  - May need -lsocket -lnsl for networking

HP-UX:
  - Native cc requires -Aa for ANSI mode
  - Add -D_HPUX_SOURCE for compatibility
  - Use GCC for easier build

SGI IRIX:
  - Native cc has excellent ANSI support
  - Use -ansi -pedantic for strict checking
  - GCC also works well

IBM AIX:
  - xlc requires -qlanglvl=ansi for ANSI mode
  - Add -D_ALL_SOURCE for full headers
  - GCC 2.x preferred

BSD Systems:
  - Native cc usually sufficient
  - May need -traditional for older BSD
  - GCC recommended for consistency

CROSS-COMPILATION FOR DOS
================================================================================

Once built on UNIX, the executable can generate MS-DOS .EXE files for
distribution to DOS users:

1. Build on UNIX:
   $ make

2. (Optional) Create DOS-compatible executable:
   Not needed! The def88186 compiler generates DEF88186 assembly (.asm)
   files, not x86 executables. The .asm files are plain text.

3. Transfer to DOS:
   $ ftp dos-machine
   ftp> bin
   ftp> put def88186 def88186.exe
   ftp> quit

4. Use on DOS:
   C:\> def88186 test.c -o test.asm
   C:\> cpuasm test.asm test.bin

TROUBLESHOOTING
================================================================================

"parser.c: undefined symbol yytype_uint8":
  Solution: Your system doesn't have stdint types.
  Add to compat.h:
    typedef unsigned char yytype_uint8;
    typedef unsigned short yytype_uint16;
    typedef short yytype_int16;

"Cannot find -lfl":
  Solution: Flex library not needed with hand-written lexer.
  Remove -lfl from linker flags.

"macro expansion too long":
  Solution: This only affects Turbo C 2.01 on DOS!
  All UNIX compilers (GCC, native cc) handle parser.c fine.

"undefined reference to getpid":
  Solution: UNIX has getpid() in unistd.h natively.
  No changes needed.
